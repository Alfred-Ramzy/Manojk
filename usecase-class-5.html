Library Management System

Actors: Member (active), Librarian (active), Administrator (active), Online Payment Gateway (passive)

Question 1
Library Management System

Read the following case study and then draw the suggested Use Case diagram and Class diagram. Please mention any pre-conditions or assumptions that you want to consider.

A university library needs an automated system to manage book records, member borrowing, reservations, fines, and librarian services. Members should be able to search for books, borrow available books, reserve unavailable books, view their borrowing history, and pay fines. To access the system, members should register for an account, in which each member is given: memberID, name, address, phoneNumber, email. Then, members can log on to the system and search for books by title, author, or category.

Each borrowed book is recorded with borrowID, borrowDate, dueDate, returnDate, status. Members can return borrowed books, and if they delay returning a book, a fine is generated. Members can pay fines physically at the library desk or online through an online payment gateway.

According to Librarians, they need access to book records and member borrowing activities. Each librarian has librarianID, name, contactInfo, shiftTime. Librarians log in to access their work dashboard in the system. Librarians can add new books, update book information, remove damaged books, and manage reservations. Each book has bookID, title, author, category, availabilityStatus.

Furthermore, Administrators have adminID, name, role onto the system to manage employee records, generate reports about borrowed and overdue books, and manage fine policies. Reports should include information about book circulation and member activities.

Use Case Diagram
Class Diagram
Assumptions & Pre-conditions: Members must register before accessing the system. Login is a pre-condition (not modeled as include). Fines are auto-generated when returnDate exceeds dueDate. Payment can be physical or online. Librarians and Admins have separate logins with different dashboards.
Class Diagram Notes: Member can have many BorrowRecords and Fines. Librarian manages Books and Reservations. Admin manages system-wide settings. Payment can be OnlinePayment or PhysicalPayment (generalization).

Online Shopping System

Actors: Customer (active), Seller (active), Administrator (active), Visa Payment System (passive)

Question 2
Online Shopping System

Read the following case study and then draw the suggested Use Case diagram and Class diagram. Please mention any pre-conditions or assumptions that you want to consider.

An online shopping company needs an automated system to manage customers, products, orders, payments, and inventory. Customers should be able to browse products, add items to a shopping cart, place orders, view order history, and pay for their purchases. To access the system, customers should register for an account, in which each customer is given: customerID, name, address, phoneNumber, email. Then, customers can log on to the system and shop for products.

Each order is given orderID, orderDate, totalAmount, orderStatus. Customers may cancel an order before shipping. Customers can pay for their orders either by cash on delivery or online through a Visa payment system.

According to Sellers, they need access to product records and order details. Each seller has sellerID, name, shopName, contactInfo. Sellers log in to access their dashboard in the system. Sellers can add products, update product prices, manage stock, and view customer orders. Each product has productID, productName, price, quantityInStock, category.

Furthermore, Administrators have adminID, name, role onto the system to manage user accounts, approve sellers, generate reports on sales operations, and monitor product availability. Bills and invoices should be generated for each order and stored in the system.

Use Case Diagram
Class Diagram
Assumptions: Customers must register and login before shopping or placing orders. Unregistered users can only browse. Cancel Order only possible before shipping. Bills are auto-generated after each order.

Actors: Student (active), Instructor (active), Administrator (active), Online Payment System (passive)

Question 3
University Course Registration System

Read the following case study and then draw the suggested Use Case diagram and Class diagram. Please mention any pre-conditions or assumptions that you want to consider.

A university needs an automated system to manage student registration, course enrollment, instructors, and fee payments. Students should be able to register for an account, log on to the system, enroll in courses, drop courses, view their schedules, and pay tuition fees. Each student is given: studentID, name, age, address, contactInfo.

Each course registration is recorded with registrationID, semester, registrationDate, status. Students may cancel their registration within the allowed add/drop period. Students can pay their tuition fees physically at the finance office or online by Visa payment through an online payment system.

According to Instructors, they need access to course rosters and student academic records. Each instructor has instructorID, name, specialization, contactInfo. Instructors log in to access their work dashboard in the system. Instructors can view course schedules, view enrolled students, and update grades for students. Each course has courseID, courseName, creditHours, schedule, classroom.

Furthermore, Administrators have adminID, name, role onto the system to manage course offerings, assign instructors, manage departments, generate tuition bills, and generate reports on course enrollment and academic performance. Each bill has billID, amount, paymentStatus, paymentDate.

Use Case Diagram
Class Diagram
Assumptions: Students must register before accessing the system. Drop is only allowed within the add/drop period. Payment can be physical at finance office or online by Visa. Instructor grades are submitted per course per semester. Admin manages course offerings and instructor assignments.

Actors: Guest (active), Receptionist (active), Hotel Administrator (active), Online Payment System (passive)

Question 4
Hotel Reservation System

Read the following case study and then draw the suggested Use Case diagram and Class diagram. Please mention any pre-conditions or assumptions that you want to consider.

A hotel needs an automated system to manage guest records, room reservations, payments, and staff management. Guests should be able to register for an account, log on to the system, search for available rooms, reserve rooms, cancel reservations, view booking history, and pay bills. Each guest is given: guestID, name, address, contactInfo, nationality.

Each reservation is given reservationID, checkInDate, checkOutDate, roomType, reservationStatus. Guests can cancel a reservation before the check-in date. Guests can pay for their bookings during their stay at the hotel or online through a Visa payment option connected to an online payment system.

According to Receptionists, they need access to guest bookings and room availability schedules. Each receptionist has employeeID, name, shiftTime, contactInfo. Receptionists log in to access their work dashboard in the system. Receptionists can confirm reservations, check in guests, check out guests, and update room statuses. Each room has roomID, roomNumber, type, pricePerNight, availabilityStatus.

Furthermore, Hotel Administrators have adminID, name, role onto the system to manage employee records, manage room categories, generate guest bills, and generate reports on hotel occupancy and revenue. Each bill has billID, amount, paymentStatus, paymentDate.

Use Case Diagram
Class Diagram
Assumptions: Guests register and login before reserving. Cancellation only before check-in date. Payment can be on-site or online via Visa. Receptionist confirms reservations and manages check-in/out. Admin manages room categories and generates bills.

Question 5
Clinic Appointment System

Read the following case study and then draw the suggested Use Case diagram and Class diagram. Please mention any pre-conditions or assumptions that you want to consider.

A medical clinic needs an automated system to manage patient records, doctor appointments, prescriptions, and billing. Patients should be able to register for an account, log on to the system, schedule appointments with doctors, cancel appointments, view medical history, and pay consultation bills. Each patient is given: patientID, name, age, address, contactInfo.

Each appointment is given appointmentID, date, time, status. Scheduled appointments can be cancelled before the appointment time. Patients can pay their consultation bills during their visit to the clinic or online through a Visa payment option using an online payment system.

According to Doctors, they need access to patient medical records and daily appointment schedules. Each doctor has doctorID, name, specialization, contactInfo. Doctors log in to access their dashboard in the system. Doctors can view appointments, access patient records, write prescriptions, and update consultation notes. Each medical record has recordID, diagnosis, prescription, visitDate.

Furthermore, Reception Administrators have adminID, name, role onto the system to manage doctor schedules, generate patient bills, manage clinic staff, and generate reports on clinic operations. Each bill has billID, amount, paymentStatus, paymentDate.

Use Case Diagram
Class Diagram
Assumptions: Patients must register and login before scheduling. Appointment cancellation only possible before appointment time. Payment can be during clinic visit or online via Visa. Doctors access patient records to write prescriptions. Admin manages doctor schedules and generates bills.
